# Architectural Decisions & Technical Choices

- **[Architecture] 3-Layer Memory Architecture (2026-04-22):** Adopted to persist state and optimize token usage via constitutional context (based on the Jules/Claude-Code architecture).
- **[Architecture] Strict Vanilla Stack (2026-04-15):** Exclusive use of HTML/CSS/JS (Vanilla). Frameworks like React or Tailwind are banned to ensure maintainability, no build steps (compilation), and maximum "standalone" portability (100% offline compatibility).
- **[Design] Glassmorphism Design System (2026-04-10):** Chose the Glassmorphism aesthetic (semi-transparent backgrounds, backdrop-filters) to provide a modern, "premium," and engaging interface for students. All colors are managed via `shared.css` variables.
- **[Design] High Contrast Mode:** A 3-state theme cycle includes a High Contrast mode (`body.high-contrast`) focusing strictly on accessibility: pure black/white elements, thick borders, and removal of subtle backgrounds, inline styles, and box-shadows.
- **[PWA & Offline] LocalStorage (2026-04-18):** The `localStorage` API is the sole persistence mechanism for settings (theme, sound), student lists, and scores, avoiding local database complexity while maintaining offline functionality.
- **[PWA & Offline] Service Worker (2026-04-30):** The project integrates a single registration point for `sw.js` via `assets/js/theme.js`, using path-agnostic logic. A "Stale-While-Revalidate" strategy automatically updates cached static assets without manual cache versioning. The fetch/update is wrapped in `event.waitUntil()` to keep the worker alive. *Note: Adding new applications requires running the `node scripts/generate-sw-manifest.js` script to automatically update the `ASSETS` array in `sw.js`.*
- **[Assets] Strict Local Hosting (2026-04-12):** No CDN dependencies are tolerated. All fonts (Outfit, Inter) and scripts must be hosted locally.
- **[Assets] Image Optimization:** Large top-down map images (`assets/img/mats/`) are optimized using Python's Pillow library: converted to adaptive 256-color palette (`P` mode) and saved with `optimize=True` to preserve hard lines (PNG) while reducing file size.
- **[Assets] FontAwesome 7 Pro (2026-04-30):** Replaced Lucide with FontAwesome 7 Pro (Solid and Duotone only). A script (`meta/scripts/generate_fa_subset.js`) dynamically extracts the necessary icons. Never cache icon DOM nodes before modification (they are replaced by `<svg>` tags).
- **[Pedagogy] Gamified Metaphors:** Exercises for children prioritize immersive, gamified mechanics (e.g., cracking a safe, physical buttons) over generic instructions, to make the experience engaging and realistic.
- **[Pedagogy] Adaptive Difficulty (2026-04-25):** Driven by the centralized `ScoreManager` module, which triggers a global `c2_change_difficulty` event. Standard levels are represented by FA circle icons (green=Facile, yellow=Moyen, red=Difficile) and fire icons (Extrême). For level-based adaptive difficulty instead of per-item streaks, override the `ScoreManager.checkAdaptiveDifficulty` check during init and manually trigger difficulty changes. New applications or game modes tracking stats must be explicitly mapped to a French translation string in the `MODE_LABELS` object within `assets/js/scores.js` to ensure the correct localized title.
- **[Pedagogy] State-Level Constraints (2026-04-24):** Pedagogical rules (forbidden moves, mandatory obstacles) are enforced at the state generation level, not the UI level, ensuring consistency regardless of the student's input method.
- **[Pedagogy] Binary Display (2026-04-25):** In binary calculation visualizations, inactive bits ("0") must always be explicitly displayed (e.g., `128 + 0 + 32 + 0 = 160`) to reinforce weight learning.
- **[Pedagogy] URL Constraints & Overrides:** Applications support specific URL parameters to enforce pedagogical constraints via `assets/js/url-params.js`. Examples include: `hideDict` in binaire_message.html, `lockTopology` in routage_reseau.html, `strictMode` in bit_de_parite.html, `hideGrid` in simulateur_bluebot.html, `unlockEditor` in binaire_studio.html, `blindcode=1` (force hiding commands), `noCmdToggle=1` (hide initially), `lockSkin` (disable skin unlock progression), `unlockAllSkins` (unlocks all skins), and `coloredCmds` (applies crane game colors to commands). When adding new toggle options to the Share Modal, the checkbox ID must strictly follow the `opt-[paramName]` naming convention.
- **[Pedagogy] Application Modes & Mechanics:**
  - **Blue-Bot Simulator:** Features 5 main modes (Exploration, Simulateur, Pilotage, Décodage, Dessin). The 'Memory' mode (toggled via `bb_memory_mode`) overrides standard mat content generation to create randomly shuffled pairs; mat types not relying on explicit content arrays (like `none`, `custom`, `city`) must explicitly bypass this logic. Item collection logic (`collectMode`) must explicitly support dynamically generated content (`config.baseContent`) in addition to static arrays (`config.content`). The mat configuration is centrally defined in `MAT_CONFIG` within `assets/js/bluebot/bluebot-skins.js`. The robot's orientation is strictly tracked as an integer (0-3). When a program execution loop is asynchronously halted (e.g., setting `simState.stopped`), the visual state and logical internal position state (like `robotRow`/`robotCol`) must be manually synchronized to prevent subsequent run desynchronization.
  - **Pixel Studio (`binaire_studio.html`):** The '#grid-editor' mode maintains a constant physical grid size regardless of cell count by applying 'flex: 1' to its rows and cells. The Editor mode ('Éditeur') remains locked and disabled by default until 3 'decode' and 3 'encode' challenges are completed. When pasting into the 'Éditeur', whitespace must be stripped before calculating dimensions.
  - **Codage binaire (`binaire_codage.html`):** The mini-calculator state (`calcState`) and vault state (`vaultState`) are independently managed and must be manually synchronized with their respective cached UI elements. The calculator is implemented as a full-page modal overlay (`.ui-modal-overlay`) to ensure it fits perfectly on narrow mobile screens and prevents simultaneous viewing of the calculator and vault, managing cognitive load.
  - **Bit de Parité (`bit_de_parite.html`):** Grids use a 1D DOM structure where cell indices map to `row * (SIZE + 1) + column` to account for the extra parity row and column.
- **[UX] Top-Tabs Navigation (2026-05-01):** Standardization of mobile and desktop navigation using exclusively top tabs (`.tabs`), eliminating bottom tab bars (`.nav-bar` is deprecated) to prevent conflicts with mobile OS gestures.
- **[UX] Embedded Controls (2026-05-01):** Action buttons within dynamic list elements must follow the document flow (relative Flexbox); the use of `position: absolute` is banned to avoid unexpected overlaps during expansion.
- **[Export] Image Export via SVG:** Exporting HTML elements as images is done by constructing an SVG string with `<foreignObject>`, drawn to a `<canvas>`, and downloaded. The SVG must be encoded as a data URI (`data:image/svg+xml;charset=utf-8,`) to avoid tainted canvas errors.
- **[Export] Clipboard API:** 'Copy to clipboard' functionality writes both `text/plain` and `text/html` (with inline styles) using the `ClipboardItem` API, allowing users to paste structured tables into Word/Excel.
- **[Security] XSS Mitigation (2026-04-25):** The use of the `.innerHTML` property is strictly regulated and to be avoided for any user-managed data. Favor `document.createElement`, `textContent`, and use `ScoreManager._escapeHtml()`.
- **[Testing & CI/CD] Playwright E2E (2026-04-30):** Code verification relies entirely on Playwright E2E test scripts located in `meta/e2e_tests/` (run via GitHub Actions). There are no unit test suites (e.g., no standard pytest/npm test) for the frontend code.
- **[Dev] Clean Production Code:** Ensure production files (`sw.js`, `index.html`) are free of `console.log` and `console.error` debug statements to avoid polluting the browser console.
- **[Dev] Repository Cleanliness:** Strict ban on committing temporary work files, drafts (`plan.md`, `test_*.py` outside the e2e folder, `server.log`), or cache folders (`__pycache__`). Modifications are made via `write_file` or precise Git diffs (the `replace_file_content` tool is refused).
